## 할일
- 만약 다른 컴퓨터(A)에서 ssh로 현재 컴퓨터(B)에 접속해서 dev-team을 실행했다면 여기서 진행되는 개발과정을 현재의 monitor 스킬을 이용해서 A 컴퓨터에서 대시보드를 볼 수 있나?

  **답변:** 직접은 안 됨. 현재 `monitor-server.py`는 `127.0.0.1`에만 바인딩하도록 하드코딩되어 있고(주석에 "0.0.0.0 is prohibited per PRD §4.1"), 네트워크 접속 옵션이 없어 A에서 B의 포트로 바로 연결할 수 없음 (scripts/monitor-server.py:6917, scripts/monitor-server.py:6903).

  해결 방법:

  1. **권장 — SSH 포트 포워딩** (설정 변경 없음, 가장 안전)
     ```
     # A에서 B로 접속할 때
     ssh -L 7321:127.0.0.1:7321 user@B
     ```
     그 뒤 A의 브라우저에서 `http://localhost:7321` — SSH 터널을 타고 B의 로컬 서버에 붙음. B에서 평소처럼 `/dev-monitor`만 돌리면 됨.

  2. **비권장 — 바인드 주소 변경**: `monitor-server.py:6917`의 `"127.0.0.1"`을 `"0.0.0.0"`으로 고치면 LAN 노출 가능하지만, PRD §4.1이 이를 명시적으로 금지하는 이유는 대시보드가 인증 없이 tmux pane 출력·WBS 상태·파일 내용을 평문으로 서빙하기 때문. 같은 네트워크의 누구나 열람 가능해지므로 SSH 포워딩이 훨씬 안전함.

  향후 개선 옵션: `/dev-monitor`에 `--bind` 옵션을 PRD 제약 내에서 추가(예: SSH 터널 전용으로 `::1` IPv6 허용)하는 설계 검토.
